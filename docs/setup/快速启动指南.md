## 🐛 常见问题

### Q1: 后端启动失败？

**检查**：
1. MySQL 是否启动？
2. 数据库 `workflow` 是否存在？
3. 端口 9099 是否被占用？

**解决**：
```bash
# 检查 MySQL
mysql -u root -p1qaz2wsx -e "SHOW DATABASES;"

# 检查端口
lsof -i:9099

# 创建数据库（如果不存在）
mysql -u root -p1qaz2wsx -e "CREATE DATABASE IF NOT EXISTS workflow;"
```

---

### Q2: 前端启动失败？

**检查**：
1. Node.js 是否安装？
2. 依赖是否安装？

**解决**：
```bash
cd frontend

# 安装依赖
npm install

# 清除缓存重新安装
rm -rf node_modules package-lock.json
npm install

# 启动
npm run dev
```

---

### Q3: 登录失败？

**检查**：
1. 后端是否启动？
2. 网络请求是否正常（F12 → Network）
3. 控制台是否有错误？

**验证后端**：
```bash
curl http://localhost:9099/api/process/health
```

应返回：
```json
{"code":200,"message":"服务正常","timestamp":...}
```

---

### Q4: 页面显示空白？

**检查**：
1. 浏览器控制台错误（F12）
2. 路由是否正确
3. Token 是否有效

**清除缓存**：
- F12 → Application → Storage → Clear site data
